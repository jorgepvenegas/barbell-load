STATE MANAGEMENT PATTERN
─────────────────────────────────────────────────────────────────────────────
AppContext Pattern:
  1. useReducer for state updates
  2. Context.Provider wraps app
  3. Custom hook (useAppStore) provides access
  4. Actions: setWeight, setPercentage, setBarWeight, etc.
  5. Auto-persists to AsyncStorage on change

ThemeContext Pattern:
  1. useState for light/dark toggle
  2. AsyncStorage for persistence
  3. Custom hook (useTheme) provides access
  4. Automatic theme switch on mount (respects system preference)
